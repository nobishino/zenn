## Go言語を完全に理解したい人が読むべきもの

Go言語を「完全に」理解したい人が読むべきドキュメントは、2つあります。

1つめは [The Go Programming Language Specification](https://go.dev/ref/spec)、つまりGoの言語仕様書です。Go言語の文法や意味論のほとんどはここに書かれています。

しかし、言語仕様書だけではGoプログラムの振る舞いを完全には説明できません。そこで2つめのドキュメントとして [The Go Memory Model](https://go.dev/ref/mem) があります。これは、複数のgoroutineが絡む「並行処理」におけるプログラムの振る舞い、特にメモリーの読み書きについて規定するドキュメントです。

つまり、The Go Memory Modelは「並行処理」について規定するドキュメントである、というのがこの章で押さえてほしいポイントです。

## ※メモリーモデルは読まないといけないのか

先に断っておくと、通常の開発上は、メモリーモデルを全く読む必要がありません。

- むしろ、読む必要がない状態が望ましい状態です

なので、興味がない人は読まなくて良いです。

- 読まなくて良いですが、並行プログラムのテストを実行するときに`-race`オプションをつけましょう
- 参考: [Data Race Detector - The Go Programming Language](https://go.dev/doc/articles/race_detector)

興味のある人・Go言語を「完全に」理解したい人は、ここから一緒に頑張っていきましょう！

## 発表の流れと得られるもの

この発表では、まず第2章で並行処理の難しさをクイズを通して体験し、私たちが普段無意識に使っているメンタルモデル（逐次一貫モデル）が破綻する例を見ます。次に第3章で、メモリーモデルの中心概念である「観測可能性」と「happens-before関係」を図解で正確に理解し、第2章のクイズを正しく解けるようになります。第4章では、Go 1.19のメモリーモデルアップデートで記述が追加されたsync/atomicパッケージについて学びます。

読み終わる頃には、The Go Memory Modelを自力で読み解くための土台が手に入っているはずです。
